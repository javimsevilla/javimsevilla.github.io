What are CSS Pre-processors?

CSS Pre-processors are compiled computer languages that extends CSS language

CSS on Steroids

"They add new functionality to help you maintain your CSS files. You write a simple code that is similar to plain CSS with some extra language constructions, compile it and the result of the compilation is a new CSS file."

Reference: http://goo.gl/2IBgL7

Why Pre-processing CSS?

Productivity

Write CSS faster

Maintenance

Less code = Easier code

How can I use CSS Pre-processors?

It depends on pre-processor vendor

How to use LESS?

  1. Install node.js (http://nodejs.org/)

  2. Install less via npm (Node Package Manager)

    $ npm install -g less
  3. Create some LESS code (style.less)

  4. Compile file:

    $ lessc style.less > style.css

How to use SASS?

  1. Install Ruby (http://rubyinstaller.org/)

  2. Install SASS gem 

    $ gem install sass

  3. Verify installation asking for sass version

    $ sass -v
  4. Create some SASS code (style.scss)

  5. Compile file:

    $ sass input.scss output.css
    $ sass --watch input.scss:output.css
    $ sass --watch app/sass:public/stylesheets

How to use STYLUS?

  1. Install node.js (http://nodejs.org/)

  2. Install Stylus via npm

    $ npm install stylus -g
  3. Create some Stylus code (style.styl)

  4. Compile file:

    $ stylus --compress < some.styl > some.css

Mixins


.headers (@font_size, @color){
  font-size: @font_size;
  color: @color;
}

h1 {
  .headers(2em, blue);
}

h2 {
  .headers(1.5em, red);
}
						        

h1 {
  font-size: 2em;
  color: blue;
}
h2 {
  font-size: 1.5em;
  color: red;
}

Mixins


@mixin headers ($font_size, $color) {
  font-size: $font_size;
  color: $color;
}

h1 {
  @include headers(2em, blue);
}

h2 {
  @include headers(1.5em, red);
}

Loops


.loop(@counter) when (@counter > 0){
    .loop((@counter - 1));

    .border-@{counter} {
        border: 1px * @counter solid blue;
    }
}

span {
  .loop(3);
}
